# V1 Scope

## Purpose of V1

The purpose of V1 is not to prove theology, build the full business, or finalize the knowledge system.

The purpose of V1 is to validate the interaction and the end-to-end experience of one decision session.

## What V1 must prove

V1 must prove that a real user can move through the entire Kairos journey:

- enter with ambiguity
- define the decision
- stay engaged through the process
- reach clarity
- close in a decision
- commit to action

## What V1 does not need yet

V1 does not need:

- final Bible licensing
- full doctrinal infrastructure
- advanced analytics
- complete payment optimization
- perfect visual design
- every use case

## Practical V1 scope

The narrowest valid V1 includes:

- one guided session format
- a single interface that supports all six phases
- the free-to-paid transition after phase 1
- code-based activation for testing and community distribution
- enough session persistence to finish one flow

## Why V1 should stay narrow

The project has large ambition, but broad scope too early will distort learning.

The first implementation must validate:

- process
- interaction
- clarity
- completion

before attempting to validate:

- scale
- doctrine systems
- multi-case complexity
- deeper automation

## Immediate strategic use of V1

V1 should be placed in the hands of:

- trusted leaders
- pastors
- close community members
- people already facing real decisions

The goal is real-world testing, not abstract approval.
