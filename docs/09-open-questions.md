# Open Questions

This project already has meaningful foundations, but several decisions should remain explicitly open.

## Spiritual and doctrinal questions

- What level of biblical quoting is acceptable in V1 before licensing?
- How should Kairos behave when biblical interpretation is highly sensitive or contested?
- When should Kairos encourage pastoral escalation instead of continuing the guided flow?

## Product questions

- How broad should the first live pilot be after the initial V1 proves the core loop?
- What is the right balance between open-text interaction and constrained selection in real user behavior?
- What additional session memory becomes necessary after the first implementation proves useful?

## User experience questions

- What tone calibration works best in real usage between firmness and gentleness?
- How long can a session pause before re-entry drops too much?
- What form of return reminder, if any, should be introduced later?

## Commercial questions

- Is `$9` the right first-session price for impulse adoption and perceived seriousness?
- What code distribution model creates the best learning without training the market to expect free access?
- At what point, if any, does Kairos expand beyond single-session monetization?

## Validation questions

- What percentage of users should reach Phase 5 before the flow is considered healthy?
- What percentage of users actually execute the action they commit to?
- What kind of decisions produce the strongest recurrence?

## Strategic principle for open questions

These are not blockers to beginning.

They are areas to validate in sequence, after the foundations remain clear and the first real usage begins.
